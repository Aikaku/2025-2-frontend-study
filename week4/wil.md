### 3. 정리

**1) React의 탄생**

- React는 복잡한 웹 화면을 효율적으로 업데이트하기 위해 만들어진 프론트엔드 라이브러리이다.  
- 기존 웹은 페이지의 일부분만 바뀌어도 전체를 다시 렌더링해야 했으나 
  React는 Virtual DOM을 활용해 변경된 부분만 빠르게 갱신한다.  
- 또 화면을 작은 단위로 나눈 컴포넌트 구조를 도입해  
  유지보수성과 재사용성을 높였다.  
- 결과적으로 React는 성능 향상, 코드 관리 용이성, 개발 효율성을 동시에 제공한다.

---

**2) Counter App을 만들어요**

- React에서는 화면의 변화가 상태에 의해 결정된다.  
- 단순한 변수 변경만으로는 화면이 갱신되지 않고 
  React가 감지할 수 있는 상태 변경이 일어나야 다시 렌더링된다.  
- 상태를 관리하기 위한 도구가 바로 useState이다.  
- 버튼 클릭 시 상태가 바뀌면, React는 Virtual DOM을 통해 필요한 부분만 업데이트한다.  

---

**3) 컴포넌트의 생애**

- React의 컴포넌트는 생성, 업데이트, 제거의 생애주기를 가진다.  
- 과거 클래스형 컴포넌트에서는 생명주기 메서드를 직접 제어했지만
  지금은 함수형 컴포넌트와 Hook을 사용해 훨씬 간결하게 관리할 수 있다.  
- 특히 useEffect를 사용하면 컴포넌트가 처음 렌더링될 때 
  혹은 특정 상태가 바뀔 때 동작을 지정할 수 있다.  
- React는 이러한 구조를 통해 개발자가 명시적으로 렌더링을 제어하지 않아도  
  자연스럽게 컴포넌트의 생애를 관리해준다.
