## <프론트엔드 스터디 5주차 WIL>

### 1. 이번주 스터디 목표
* useEffect를 활용해 서버와 통신해볻나.  
* 데이터를 받아와 화면에 렌더링하고 컴포넌트 단위로 코드를 작성한다.

---

### 2. 오늘의 키워드
* useEffect, fetch, API, useState, Component, Props, 리팩토링

---

### 3. 정리

**1) 서버와 통신하기**

- React에서 서버와 데이터를 주고받기
  - `fetch()` 함수를 이용해 외부 API에서 데이터를 요청하고 응답으로 받은 JSON 데이터를 상태에 저장해 화면에 보여줌
  - 단순히 정적인 화면이 아닌 실제 데이터를 주고받는 동적인 웹임

---

**2) API를 받으면 컴포넌트는 어떻게 알죠?**

- React는 DOM을 직접 바꾸지 않고 Virtual DOM을 이용해 효율적으로 렌더링함
- 하지만 단순히 데이터를 받아온다고 해서 화면이 바로 갱신되지는 않음 
- 그래서 `useState`로 상태를 관리하고, `useEffect`로 데이터 요청 시점을 제어함 
- `useEffect`의 의존성 배열을 이용하면 컴포넌트가 처음 렌더링될 때 한 번만 API를 요청하도록 만들어 무한 반복을 방지 가능
- 즉, useState는 “값의 변화”를, useEffect는 “언제 실행할지”를 관리하는 역할임

---

**3) 컴포넌트로 리팩토링**

- 하나의 파일에 모든 코드를 넣는 대신 역할별로 컴포넌트를 분리했음
- 데이터가 없을 때 화면들을 각각의 컴포넌트로 나누어 관리하니 훨씬 구조가 보기 좋아졌다, 
- 이때 상위 컴포넌트에서 데이터를 props로 전달해 하위 컴포넌트가 재사용될 수 있도록 했다.  
- React의 핵심은 결국 “데이터의 흐름과 구조화된 컴포넌트”
